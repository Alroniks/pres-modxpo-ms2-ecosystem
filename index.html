Required elements of shop

Categories and products
List of categories, list of products, multicategories, sorting by price, list of brands, sorting by brand, related products, product page, rating and reviews...
Cart and orders
Cart block on pages, checkout page, order details, email notifies, chat notifies, statuses of order...
Deliveries
Address details, user details, additional cost depending on sizes, weight, distance, should have ability customize logic...
Payments
Additional cost if laws allow, customisible logic of processing...
Options and
characteristics
Abilty define unlimited list of options and characteristics, that can be used for search and filtering...

Minishop. First generation

Released on 28th of Febrary, 2012
as alternative existing VisionCart and Shopkeeper

  • Categories, multicategories and products
    (via system settings and templates)
  • Warehouses (no more in version 2)
  • Cart, orders, order statuses
  • Payment not extensible

Mini Shop 2

Mini Shop 2

  • Released in a year after first version: March 3, 2013
  • Warehouses was removed
  • Categories and Products as CRC
  • Cart, Orders, Statuses was rewritten
  • Was added Deliveries and Payments, also Gallery
  • Relations between products, Customisible Options

Ecosystem

Store vs Marketplace

  • One developer or team
  • No way to add extra for sale
  • Support from team
    (best quality)
  • Quality of extras
  • Many independent developers
  • Easy to add own extra
  • Support from developer, no guarantee
  • Fee or percent to marketplace owner
  1. Official MODX Repository - market place
  2. modstore.pro

Existing stores

  • modmore.com - Mark Hamstra and team
  • extras.io - Adido Team
  • rstore.pro - Vladimir Grishin
  • aestore.by - me, wip

I hope in future each store will growth to real marketplace.
It's have to impact, but give awesome results.

History of modstore.pro

  • Started on 11th of June, 2013
  • store.simpledream.ru at first
  • Basically extras from staff
  • Fully based on MODX, include UI for manage packages and versions

How it works?

  1. Developer send Extra to Store
  2. Store check and validate developer and Extra
  3. Store publish Extra and provide payment tools
  4. After developer provide support to customers throught Store

For paid extras average commision is 30%.
Store guarantee support only for paid extras.

How it create?

  1. Create or setup repository for MODX packages
  2. Build private section users with ability create auth keys
  3. Set content managment process: descriptions, documentation, tutorials, logos
  4. Build support section
  5. Built billing section

How it support?

  • Hire designer for make logos
  • Hire people for provide support or require support from authors with penalties if not
  • Hire writer for create descriptions
  • Require authors publish documentation before package
